Business Requirements Document

Objective: Build a centralized sales management system to organize workflows, prevent lead leakage, and structure client data accurately based on the unique B2B conformity assessment model.

1 Foundational Data Model & Identity Management

The system is built on a "Conformed Data Model" where the Commercial Registration (CR) Number is the unique identifier for Facilities.

Architecture Visualization

This diagram illustrates the structural entity relationships and the flow from Lead conversion to Deal creation linking Facilities and Brokers.

1.1 Structural Entity Relationships

Core Entity
🏢

Facility

The manufacturer/plant receiving the certificate. Inherits from Company table.

🚨 CR Number MUST be unique.
Linked to one or many Deals. Created automatically when a Lead reaches "Qualified" stage.
Payer Entity
🤝

Client / Broker

The payer/communicator. Can be linked to multiple Facilities.

Mandatory Toggle (Yes/No)

Intake Entity
🎯

Lead

Initial prospect.

Converted into Deal + Facility + Client upon reaching the "Qualified" stage.
Transactional Entity
💰

Deal / Opportunity

The transactional record.

Relationship Rule: Must link to exactly one Facility and one Client/Broker side-by-side.
Multiplicity Rule: Multiple deals can exist for the same Facility (CR Number) simultaneously.
Log Entity
📝

Activity

Logged interactions.

Must belong to a parent Deal/Lead.

2 Sales Pipeline & Kanban Workflows

The CRM features a strict visual Kanban board. Data validation is enforced at specific stage gates to ensure data integrity before a deal can progress.

3.1 Three Distinct Entry Workflows

Deals enter the pipeline at different stages based on their source. The system must support the following automated entry points:

1. Website Capture

Leads captured directly from the corporate website forms.

➡️ Lands in "New"
2. Manual / Marketing

Leads uploaded by Marketing or manually added by reps.

➡️ Lands in "Contacted"
3. Renewals (Conditional)

After-Sales deals. Routing depends entirely on historical data completeness.

✅ Data Exists ➡️ Negotiation ❌ Missing Data ➡️ New

3.2 Pipeline Stages & Enforcement Logic

1
New

Entry Point: Website leads OR Renewals missing historical data.

Action (Depends on Source):
▪️ If Website: Rep selects activity (Call/WhatsApp), logs interaction, writes initial feedback.
▪️ If Renewal: Rep must urgently contact the client to gather the missing Facility/Client data before progressing.

2
Contacted / Verification

Entry Point: Marketing/Manual leads land here. Website/Incomplete Renewals move here after initial contact.

Action: Log activities and write comments regarding the previous stage. Rep must send a message to customer via WhatsApp/Email to verify data and log this attempt.

CRITICAL DB COMMIT STAGE
3
Qualified

System Action: Lead is officially converted. "Facility" and "Client" records are formally separated and stored in the database.

Validation Gate: System blocks saving unless these fields are filled: Country, Service Type, Contact Number, CR Number.

Action: Select a Tag, edit Opportunity Name to format [Company Name] - [Service Type], input Expected Revenue. Take logs and write feedback.

4
Proposal / Quotation

System Action: System allows the preparation of documents and requirements.

Validation Gate: Request Number, Authority, CR Number, Price, Currency, Expiry Date.

Action: Log the activity and write feedback.

5
Negotiation

Entry Point: Renewal deals (with complete historical data) land directly here.

Validation Gate: New Price, Currency, Expiry Date.

Action: Send formal quotation to the client. Log activity and write feedback regarding negotiation status.

6
Won 🏆

System Action: Display a "Great Job" visual animation/mark on screen. Auto-increment user's "Won Deals" counter.

Action: Log final activity.

Lost ❌
Validation Gate (Mandatory Dropdown): Financial issues, Customer not qualified, Product not compliant, Competitor, Technical, response delay, Not enough stock, Didn't Need Service.

Action: Mandatory text note detailing exact context. Auto-increment user's "Lost Deals" counter.

5 Module 4: SLA Automation & Activity Tracking

Automated background workers, escalation rules, and interactive activity planning.

5.1 Deal Neglect & Pipeline SLAs

The CRM implements a background monitoring worker checking the "Last Activity", "Assignment Date", and "Stage Change Date" on every open deal using Absolute Calendar Days.

⏱️ Intake Response SLA (24 Hours)

If a newly assigned lead receives no logged activity (first contact/action) within 24 hours of its assignment date, a "Delayed First Response" alert is sent to the Manager.

🚦 Stage Stagnation SLA (7 Days)

If a deal remains in the exact same pipeline stage for more than 7 days without moving to the next stage or being closed, an alert is triggered to prevent pipeline clogging.

General Neglect Timer (No Activity)
Day 024567+
✅

Active Deal

Activity logged recently. No SLA breaches.

5.2 Task & Activity SLAs (Reminders & Overdue)

📅 Daily Visibility & Reminders

Scheduled activities (e.g., a meeting planned for tomorrow) will automatically populate in the Rep's "Today’s Activities" widget and trigger an in-app reminder on the due date.

🔥 Overdue Task SLA

If a scheduled task/activity passes its deadline without being marked as "Done" or rescheduled, an automated escalation email is immediately sent to both the assigned Sales Rep and the CRM Manager.

5.3 Activities Type

5.4 Activities Plan

Opens an interactive calendar view, allowing Reps to schedule and visualize multiple upcoming tasks and meetings linked to specific deals.

7.1 Role-Scoped Dashboards & KPI Formulas

👑

Management Dashboard

Revenue Overview
Total Revenue Scorecard = Sum (Value of Won Deals)
Revenue by Year Bar/Line Chart = Sum (Won Deals) grouped by Year (Closed Date)
Won vs Lost Revenue Donut Chart Won = Sum (Won Deals Value) Lost = Sum (Lost Deals Expected Value)
Pipeline Health
Pipeline Value Scorecard = Sum all stages (Deal Value for stages ≠ Won & Lost)
Open Opportunities Scorecard = Count all stages (Deal Value for stages ≠ Won & Lost)
Win Rate Gauge Chart = Won Deals ÷ (Won + Lost Deals)
Retention & Risk 🚨
Neglected Deals Red Scorecard = Count (Deals with no activity ≥ 96 hours)
Neglected Revenue Red Scorecard = Sum (Deal Value where no activity ≥ 96h)
Customer Value & Performance
Top 10 Facilities (by CLV) H-Bar Chart CLV per Facility = Sum (Won Deals grouped by CR)
Average CLV Scorecard = Total Revenue ÷ Number of Unique CR Numbers
Top Sales Reps Leaderboard Table = Sum (Won Revenue per Rep) sort by Descending
Deals Closed per Rep Column Chart = Count (Won Revenue per Rep) sort by Descending
💼

Sales Dashboard

Daily Ops & Revenue
My Leads Today Scorecard = Count (Deals assigned to Current User WHERE Assignment_Date = Today)
Tasks Due Red Scorecard (If Late) = Count (Tasks assigned to Current User WHERE Due_Date <= Today AND Status ≠ Done)
Calls / Day Scorecard = Count (Activities WHERE Type="Call" AND Logged_By=Current User AND Date=Today)
Monthly Revenue Scorecard w/ Sparkline = Sum (Value of Won Deals assigned to Current User WHERE Closed_Date = Current Month)
Pipeline Performance
Total Leads = Count (Deals assigned to Current User)
Qualified Leads = Count (Deals WHERE Stage = "Qualified")
Won Deals = Count (Deals WHERE Stage="Won" AND Closed=Period)
Qualified Rate % Gauge Chart = (Qualified Leads ÷ Total Leads) * 100
Sales Conversion % Gauge Chart = (Won Deals ÷ (Won + Lost)) * 100
🚀

Marketing Dashboard

Acquisition & Conversion
Leads by Source Donut Chart = Count (Total Leads) Grouped by (Source)
Leads by Campaign H-Bar Chart = Count (Total Leads) Grouped by (Campaign)
Source Conversion % Funnel Chart = (Won Deals from Source ÷ Total Leads from Source) * 100
Revenue by Source H-Bar Chart = Sum (Value of Won Deals) Grouped by (Source)
Revenue by Campaign H-Bar Chart = Sum (Value of Won Deals) Grouped by (Campaign)
Cost Tracking & ROI
CPL (Cost Per Lead) Scorecard = Total Cost of Campaign ÷ Total Leads generated
CAC (Acquisition Cost) Scorecard = Total Cost of Campaign ÷ Total Won Deals generated
Lead Value (LV) Scorecard = Total Rev from Campaign ÷ Total Leads from Campaign
ROI % Scorecard (Red/Green) = ((Rev from Campaign - Cost of Campaign) ÷ Cost) * 100

🔗 8.2 Duplicate Merge Tool

Restriction: Sales Reps DO NOT have the ability to scan the database or merge facilities.

Tool Functionality: A dedicated tool for Managers/Admins that scans the database for matching CR Numbers or Fuzzy Names.

Actions (Pop-up offers 2 options): Save New OR Merge Old (System executes: Keep old, Merge contacts, Merge companies, Merge activities).

📤 8.3 Data Migration

Migration from legacy systems will be done via CSV or Excel import.

Pre-requisite: Data must be manually cleaned pre-upload.

Duplication Prevention: System matches existing records by CR Number during import to prevent initial duplication.

The system maintains an immutable log recording for total transparency and dispute resolution.

  • • Action Type (Create, Edit, Move)
  • • Old Value vs. New Value
  • • Timestamp

10 Detailed User Story Catalog & Epics

Developer-ready agile user stories and implementation requirements.

  • US 1.1: As an Admin, I want to create user accounts and assign branch-specific roles so that I can control data access and operational scoping.
  • US 1.2: As a User, I want a secure login and an automated "Forgot Password" link so that I can manage my access without frequent IT intervention.
  • US 1.3: As a Manager, I want to toggle a Sales Rep’s status to "On Leave", and the system MUST prompt me to select a temporary delegate, so that the absent rep's active deals are managed without delay.
  • US 2.1: As a User, I want to create a Lead with mandatory fields (Name, Phone, Service, Source) so that we maintain high data quality from the start.
  • US 2.2: As the System, I want to automatically route leads based on Country and use Round Robin for distribution, actively skipping users marked as "On Leave", so that workload is fair.
  • US 2.3: As a Sales Rep, when I enter a "CR Number" for a new facility, I want the system to instantly check the database and show a red warning with a link if it exists, so that duplicate facilities are prevented at the entry point.
  • US 2.4: As a Manager/Admin, I want a dedicated "Merge Tool" to scan for fuzzy duplicate facilities and merge their contacts and activities into a single "Keep Old" record, so that the database remains clean.
  • US 3.1: As a Sales Rep, I want to move deals through visual stages on a Kanban board so that I can manage my pipeline effectively.
  • US 3.2: As the System, when a deal reaches the "Qualified" stage, I want to formally commit the "Facility" and "Client" data to the database linking them via the CR Number.
  • US 3.3: As a Sales Rep, when creating a Quotation, I want to select a service from a dropdown and have the system auto-fetch the price from the imported service catalog.
  • US 3.4: As a Sales Rep, if I apply a discount >15%, I want the system to change the quote status to "Pending Approval" and route it to my Manager.
  • US 3.5: As a Manager, if I reject a discount, I want to be forced to enter a "Rejection Reason/Note", so that the Sales Rep knows exactly why it was rejected.
  • US 3.6: As a Sales Rep, when dragging a deal to "Lost", I want the system to force me to select a reason from a dropdown and write a mandatory note.
  • US 4.1: As a Sales Rep, I want to log Calls, Emails, and Meetings on a deal’s profile.
  • US 4.2: As the System, I want to track Absolute Calendar Days of inactivity. At 4 days, I send an alert. At 5 days, the card turns orange. At 6 days, I apply a red (⚠) SLA Breach badge.
  • US 4.3: As a Manager, I want to receive an alert if a "New Lead" has 0 logged activities within the first 24 hours of assignment.
  • US 4.4: As a Sales Rep, I want a read-only "WhatsApp View" tab inside the Deal/Lead profile fetching history via integration.
  • US 5.1: As an After-Sales Rep, I want the system to automatically trigger renewal alerts 30 days before Standard certificates expire, and 7 months before SQM certificates expire.
  • US 5.2: As a Manager, I want to receive an automated weekly summary report of pending renewals.
  • US 5.3: As an After-Sales Rep, I want a "One-Click Renew" button on expiring certificates that instantly creates a new Deal prepopulated with the CR Number, Facility, and Broker data.
  • US 6.1: (Audit Log Viewer): As an Admin, I want a 'History Log' button on every Deal to see who changed what and when, so that disputes over deal ownership or manual edits are easily resolved.
  • US 6.2: (Pipeline Search & Filters): As a User, I want a sticky search bar to find deals instantly by CR Number, Company Name, or Phone, so that navigation is seamless.

11 User Acceptance Testing (UAT) Criteria

Standardized Given-When-Then scenarios to validate system functionality before go-live.

⚖️ UAT 1: Discount Workflow

Given A rep applies a 20% discount on a quotation.

When They click 'Save'.

Then The system status changes to "Pending Approval" and routes to the Manager.

If rejected, the manager is forced to add notes. If approved, status becomes "Approved".

🛡️ UAT 2: Duplicate Prevention

Given A rep is qualifying a lead.

When They enter a CR Number that already exists in the database.

Then The system throws a warning and auto-links the new Deal to the existing Facility, preventing the creation of a duplicate record.

🏖️ UAT 3: Leave Delegation

Given A Manager sets a Sales Rep's status to "On Leave".

When The status is toggled in the system.

Then A prompt strictly requires the selection of an active Rep to inherit the open pipeline temporarily.

⏱️ UAT 4: SLA Alert & Reset

Given A deal has had no logged activity for 96 hours.

When The background worker checks inactivity.

Then An alert is sent, and the deal is marked with a warning.

If the Rep then logs a Call/Activity, the SLA timer must reset to 0 and the warning is removed.

♻️ UAT 5: Renewal Flow

Given An issued certificate is approaching its expiration date.

When It reaches the predefined threshold (30 days for Standard, 7 months for SQM).

Then An alert is triggered, and a renewal deal can be created via a single click populating existing data.

🔐 UAT 6: RBAC Enforcement

Given A user with the "Sales Rep" role logs into the system.

When They attempt to access the Global Management Dashboard or the Duplicate Merge Tool.

Then The system strictly denies access and these options are completely hidden from their navigation menu.
